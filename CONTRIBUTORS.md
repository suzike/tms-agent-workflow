## 如何贡献

本项目为公司内部项目，欢迎团队成员参与贡献。

### 开发流程

1. **Fork** 本仓库（或创建分支）
2. 基于 `main` 创建特性分支：`feat/描述`、`fix/描述`、`docs/描述`
3. 编写代码，遵循项目 [CLAUDE.md](CLAUDE.md) 中的技术约束
4. 确保 `pytest -q` 全量测试通过
5. 提交 Commit（遵循 [约定式提交](https://www.conventionalcommits.org/zh-hans/)）
6. 发起 Merge Request 到 `main`
7. 至少 1 位核心成员 Code Review 后合入

### 提交规范

```
<type>: <简短描述>

<详细说明（可选）>
```

| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 缺陷修复 |
| `docs` | 文档变更 |
| `refactor` | 重构（不改变功能） |
| `test` | 测试相关 |
| `chore` | 构建/工具/依赖 |
| `perf` | 性能优化 |
| `ci` | CI/CD 变更 |

### 代码风格

- Python 3.14 环境
- 不可变数据模式（返回 state 增量，不原地修改）
- 代码注释用中文
- 所有阈值集中在 `config.py`
- 函数 < 50 行，文件 < 800 行

### 分支策略

- `main` — 稳定版本，始终可部署
- `feat/*` — 新功能开发
- `fix/*` — 缺陷修复
- `docs/*` — 文档更新
